1. The Four Layers of Box Model

In CSS, every element is treated as a rectangular box consisting of four layers from inside to outside: Content ‚Üí Padding ‚Üí Border ‚Üí Margin

CSS Code 
/* Complete Box Model */
.box {
    /* Layer 1: Content */
    width: 180px;
    height: 40px;

    /* Layer 2: Padding (15px * 2) */
    padding: 15px;

    /* Layer 3: Border (4px * 2) */
    border: 4px solid #e74c3c;

    /* Layer 4: Margin */
    margin: 20px;
}

/*
 * Actual element width = width + padding*2 + border*2
 * Actual element height = height + padding*2 + border*2
 * margin is not included in element width/height, only affects outer spacing
 */

4.2 Horizontal Centering Essential ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

margin: 0 auto centers block elements horizontally

Centered
margin: 20px auto
CSS Code 
/* Horizontal centering - for block elements */
.center-auto {
    width: 200px;           /* ‚ö†Ô∏è Required: must set width, otherwise auto won't work */
    margin: 0 auto;         /* vertical 0, horizontal auto (centers element) */
    background-color: #27ae60;
    color: white;
    padding: 15px;
}

/*
 * How margin: 0 auto works:
 * 1. First value 0: vertical margin is 0
 * 2. Second value auto: horizontal margin is auto
 *    - Browser calculates: (parent width - element width) / 2
 *    - Left and right margins are equal, element centers
 *
 * ‚ö†Ô∏è Important Notes:
 * - Must set width for block elements (cannot be auto or unset)
 * - Element must be display: block (default for div, p, etc.)
 * - Parent container must have a defined width
 *
 * ‚ùå Wrong:
 * .center-auto { margin: 0 auto; }  /* no width, won't center */
 *
 * ‚úÖ Correct:
 * .center-auto { width: 200px; margin: 0 auto; }
 */

5. Margin Collapsing

When two block elements are stacked vertically, their vertical margins collapse into the maximum value

5.1 Sibling Margin Collapsing Essential ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

Spacing = max(margin-bottom, margin-top)

Upper Element (margin-bottom: 30px)
Actual spacing: 50px (takes maximum)
Lower Element (margin-top: 50px)
CSS Code 
/* Sibling margin collapsing */
.collapse-box1 {
    margin-bottom: 30px;
    background-color: #3498db;
    color: white;
    padding: 15px;
}

.collapse-box2 {
    margin-top: 50px;
    background-color: #9b59b6;
    color: white;
    padding: 15px;
}

/*
 * Actual spacing = max(30px, 50px) = 50px
 * NOT 30px + 50px = 80px!
 */

5.2 Solution: Trigger BFC Essential ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

overflow: hidden triggers BFC, prevents margin collapsing

Upper Element (30px)
Lower Element (50px)
CSS Code 
/* Solution: trigger BFC
 *
 * What is BFC?
 * BFC (Block Formatting Context) is an independent rendering region.
 * Elements inside BFC are isolated from external elements, preventing margin collapsing.
 *
 * Core Features of BFC:
 * 1. Prevent margin collapsing - Margins inside BFC won't merge with outside
 * 2. Contain floats - BFC includes the height of internal floated elements
 * 3. No overlap with floats - BFC area won't overlap with external floated elements
 */
.bfc-wrapper {
    overflow: hidden; /* triggers BFC */
    background-color: #f8f9fa;
    padding: 10px;
}

/* Other BFC Triggers (choose one):
 * - display: flow-root (recommended, no side effects)
 * - float: left/right
 * - position: absolute/fixed
 * - display: inline-block
 * - display: table-cell
 */

5.3 Parent-Child Margin Collapsing Essential ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

Child's margin merges with parent's margin

Parent background
Child (margin-top: 30px)
Parent top margin is also 30px
CSS Code 
/* Parent-child margin collapsing */
.parent-box {
    background-color: #27ae60;
    padding: 20px;
    margin-top: 20px;
}

.child-box {
    margin-top: 30px;
    background-color: #e74c3c;
    color: white;
    padding: 15px;
}

/*
 * Child's 30px margin-top merges with parent's margin
 * Parent top actual margin = max(20px, 30px) = 30px
 */

/* Solution 1: Add padding or border to parent */
.parent-box {
    padding-top: 1px; /* or border-top: 1px solid transparent; */
}

/* Solution 2: Trigger parent BFC */
.parent-box {
    overflow: hidden;
}

6. box-sizing Box Model Type

Recommended to use box-sizing: border-box globally

6.1 content-box vs border-box Essential ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

Same settings: width: 200px, padding: 20px, border: 5px

content-box
Total width: 250px
(width doesn't include padding/border)
border-box
Total width: 200px
(width includes padding/border)
CSS Code 
/* content-box (default) */
.content-box {
    box-sizing: content-box;
    width: 200px;
    padding: 20px;
    border: 5px solid #3498db;
    /* Actual total width = 200 + 40 + 10 = 250px */
}

/* border-box (recommended) */
.border-box {
    box-sizing: border-box;
    width: 200px;
    padding: 20px;
    border: 5px solid #9b59b6;
    /* Actual total width = 200px (includes padding and border) */
    /* Actual content width = 200 - 40 - 10 = 150px */
}

6.2 Global box-sizing Setting Essential ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

Recommended to use border-box globally in all modern projects

‚úÖ Recommended setting:

html { box-sizing: border-box; }

‚úÖ Inherit setting:

*, *::before, *::after { box-sizing: inherit; }
CSS Code 
/* Global box-sizing setting (recommended) */
html {
    box-sizing: border-box;
}

*, *::before, *::after {
    box-sizing: inherit;
}

/* Why use border-box?
 * 1. More intuitive: set width IS the actual element width
 * 2. Easier to calculate: changing padding/border doesn't affect total width
 * 3. Responsive-friendly: easier to implement percentage layouts
 * 4. TailwindCSS uses border-box by default
 */
