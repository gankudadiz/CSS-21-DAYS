Observation: moving the slider changes root font-size. The px box stays the same, while the rem box scales.

CSS 
/* Root element */
html {
    font-size: 16px; /* dynamic */
}

/* Gray box: px */
.box-px {
    width: 100px; /* always 100px */
    background: #95a5a6;
}

/* Blue box: rem */
.box-rem {
    width: 6.25rem;
    /* computed = 6.25 * html font-size */
    /* 16px * 6.25 = 100px */
    /* 20px * 6.25 = 125px */
    background: #3498db;
}

Why rem is stable Recommended ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

rem has one reference only: the root html font-size. It does not depend on parents, so it does not grow with nesting.

Quick memory: 1rem = html font-size (computed).
Why ‚Äúrelative to html‚Äù? html is the root of the DOM tree. rem reads the baseline directly from the root instead of inheriting through parents.
Example: if html is 16px, then 6.25rem = 100px; if html becomes 20px, then 6.25rem = 125px.
1. Outer (1.2rem)
2. Middle (1.2rem)
3. Inner (1.2rem) - stays the same size
CSS 
/* rem is always relative to html font-size */
.container {
    font-size: 1.2rem;
}

The ‚Äúcompound interest‚Äù effect of em Use with caution ‚≠ê‚≠ê‚òÜ‚òÜ‚òÜ

em is relative to the parent font size. With deep nesting, it compounds layer by layer.

1. Outer (1.2em)
2. Middle (1.2em)
3. Inner (1.2em)
4. Deep (1.2em) - gets huge!
CSS 
body { font-size: 16px; }

.container {
    font-size: 1.2em; /* parent x 1.2 */
    border: 1px solid #ddd;
    padding: 10px;
}

/* 
Level 1: 16px * 1.2 = 19.2px
Level 2: 19.2px * 1.2 = 23.04px
Level 3: 23.04px * 1.2 = 27.65px
Level 4: 27.65px * 1.2 = 33.18px
*/

vw and vh Demo Common ‚≠ê‚≠ê‚≠ê‚≠ê‚òÜ

vw / vh are viewport units: 1vw equals 1% of the viewport width, and 1vh equals 1% of the viewport height.

What this demo does: resize the container from the bottom-right to simulate a ‚Äúviewport‚Äù, and show how 50vw / 20vh / 5vw translate into pixels.
Common use cases: full-screen sections height: 100vh; responsive headings font-size: 5vw; modals/panels with max-height: 80vh.
Note: on mobile, browser UI (address bar/toolbars) can affect vh. For more stable behavior, consider dvh/svh/lvh.

Responsive Card (mixed units) Hands-on ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

Best practices:
1. Use % or Flex/Grid for layout width
2. Use rem for spacing and typography
3. Use px for borders and tiny details

Image

Card Title

This is a responsive card. Use rem for typography and spacing, px for borders, and let flex control the width.

CSS 
.card {
    /* Layout */
    flex: 1 1 250px;
    
    /* Details (px) */
    border: 1px solid #e0e0e0;
    border-radius: 8px;
    
    /* Spacing (rem) */
    padding: 1.5rem;
}

.card-title {
    /* Typography (rem) */
    font-size: 1.25rem; /* ~20px */
    margin-bottom: 0.5rem;
}

.card-text {
    font-size: 1rem; /* ~16px */
    line-height: 1.5;
}
