max-width vs min-width Key Point ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

max-width: Applies when screen width is less than or equal to a value (downward adaptation)
min-width: Applies when screen width is greater than or equal to a value (upward enhancement)

max-width Approach (Desktop First)
@media (max-width: 768px) { ... }
Apply styles when screen ‚â§768px
min-width Approach (Mobile First)
@media (min-width: 768px) { ... }
Apply styles when screen ‚â•768px
CSS Code 
/* Desktop first: Start from large screens, progressively hide/simplify */
.navbar {
    display: flex;
    justify-content: space-between;
}

@media (max-width: 768px) {
    .navbar {
        flex-direction: column;
    }
}

@media (max-width: 480px) {
    .navbar {
        display: none; /* Hide navigation on mobile */
    }
}

/* Mobile first: Start from small screens, progressively enhance */
.content {
    width: 100%;
    padding: 15px;
}

@media (min-width: 768px) {
    .content {
        width: 75%;
        padding: 20px;
    }
}

@media (min-width: 1200px) {
    .content {
        width: 60%;
        max-width: 800px;
    }
}

Common Media Features Useful ‚≠ê‚≠ê‚≠ê‚≠ê‚òÜ

In addition to screen width, there are other useful media features.

orientation: portrait - Portrait mode
orientation: landscape - Landscape mode
prefers-color-scheme: dark - Dark mode
prefers-reduced-motion - Reduced motion preference
CSS Code 
/* Landscape mode */
@media (orientation: landscape) {
    .banner {
        height: 200px;
    }
}

/* Dark mode support */
@media (prefers-color-scheme: dark) {
    body {
        background: #1a1a1a;
        color: #f0f0f0;
    }
    .card {
        background: #2d2d2d;
        border-color: #444;
    }
}

/* Reduced motion */
@media (prefers-reduced-motion: reduce) {
    * {
        animation: none !important;
        transition: none !important;
    }
}

Mobile First vs Desktop First Must Learn ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

Mobile First is the industry-recommended approach, start with mobile styles and progressively enhance.

Recommended: Mobile First

Characteristics:
1. Default styles for mobile
2. Less code, better performance
3. Progressive enhancement, consistent UX

Not Recommended: Desktop First

Characteristics:
1. Default styles for desktop
2. Mobile needs to override more styles
3. Easy to miss mobile adaptations
CSS Code 
/* ========== Mobile First (Recommended) ========== */

/* Base styles - Mobile */
.card {
    width: 100%;
    padding: 15px;
    margin-bottom: 10px;
}

/* Tablet and above */
@media (min-width: 768px) {
    .card {
        width: calc(50% - 20px);
        margin: 10px;
        padding: 20px;
    }
}

/* Desktop and above */
@media (min-width: 1200px) {
    .card {
        width: calc(33.333% - 20px);
        padding: 25px;
    }
}
